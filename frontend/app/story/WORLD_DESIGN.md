# タイプクエスト 8章 全体構成・マップ設計

`frontend/DESIGN.md` 7.1節の8章構成（章一覧・テーマ）を土台に、マップの作り方と
物語の流れをさらに詳細化したドキュメント。着手前にここへ書き足してから実装する、
という運用ルール（`frontend/DESIGN.md`冒頭参照）に従って、2〜8章はここに設計を
まとめておく（1章の技術的な実装詳細は [`app/story/DESIGN.md`](DESIGN.md) 側）。

## 1. マップは2種類、という考え方

### ① ワールドマップ（世界地図）＝章と章をつなぐ1枚の地図
第1章で作った `FIELD_MAP`（村の外に広がる、ドラクエ風の縮図スケールの世界地図。
[`DESIGN.md`](DESIGN.md) 4.2.1節参照）を、**章が進むごとに同じ1枚のまま拡張していく**。
新しいマップ「種類」を章ごとに作るのではなく、同じ`FIELD_MAP`のグリッドを広げ、
新しい地形（砂漠・毒沼など）や`FIELD_LANDMARKS`を足していくだけでよい。

具体的な開放の仕組み（第1章で既に実装済みのパターンをそのまま繰り返す）：

1. 次の章の入り口は、あらかじめ`FIELD_LANDMARKS`として**見えるだけ**置いてある
   （ぶつかると「まだ早い」とコトに止められる。今の2章「砂漠の町」・3章「魔法の森」の
   目印がこのパターン）。
2. 現在の章のボスを倒すと、その章の`FIELD_LANDMARKS`エントリを**`kind: "exit"`の
   本物の入り口に差し替える**（`StoryGame.tsx`の`handleBump`は既に`kind: "exit"`を
   汎用的に扱えるので、このエントリの差し替えだけで動くようになる）。
3. 新しく開放したエリアの、さらに先に見える次の章の目印を追加する（必要なら
   `FIELD_GRID_WIDTH`/`FIELD_GRID_HEIGHT`を広げる）。

つまり「ワールドマップという1つの技術的な仕組み」を1回作れば、あとは**章ごとに
データを足すだけ**で済む。新しい種類のマップエンジンを毎回作る必要が無い。

### ② エリアマップ（章ごとの本拠地マップ）＝各章の中身
各章の入り口をくぐると、その章専用の**詳細な1:1スケールのマップ**に入る
（第1章の`TOWN_MAP`と同じ考え方）。ここが実際に町を歩いたり、ダンジョンの
ギミックを解いたりする場所。章によっては「町」と「ダンジョン」で2枚に分かれる
（例：2章は「町」1枚、4章は「町的な導入部」なしで「洞窟」1枚、など章の内容に
応じて枚数を調整してよい）。

エリアマップ側の当たり判定・Yソート・徘徊NPCなどの仕組みは、`chapter1Data.ts`と
`GridExplorer.tsx`にある既存の仕組み（`buildTownTiles`パターン、`scatterObjects`、
`Interactable`/`PlacedObject`）をそのまま章ごとに使い回せる設計にしてある。

### まとめ図

```
                    ワールドマップ（FIELD_MAP、1枚を拡張し続ける）
                                    │
        ┌───────────┬───────────┬───────────┬─── …
        │           │           │           │
   1章エリアマップ  2章エリアマップ  3章エリアマップ  4章エリアマップ …
   (TOWN_MAP等)    (町+砂漠+…)   (森+神殿+…)    (洞窟)
```

## 2. 章ごとの詳細

### 第1章：はじまりの村（テーマ：チュートリアル）
→ 実装済み。詳細は [`app/story/DESIGN.md`](DESIGN.md) を参照。

### 第2章：砂漠の町（テーマ：防具購入）

**新要素**：お金（ゴールド）、防具屋・武器屋が実際に使える（1章では「棚は空っぽ」
だった建物がここで機能する）、防御力の概念がプレイヤーにも分かりやすく効いてくる。

**物語の流れ**：
- 世界地図で「砂漠の町」の入り口が開放され、村を出て初めての本格的な旅に出る。
- 町に着くと、水不足・盗賊などで困っている住人がいる（word: money/armorに関連する
  言葉を覚えるきっかけにする）。
- 防具屋の店主から「装備を整えないと、この先の砂漠もピラミッドも危ない」と言われ、
  初めて買い物をする（お金は道中の戦闘スコア・宝箱から貯める想定）。
- 謎の青年が再登場し、少しだけヒントをくれて去っていく（正体はまだ明かさない）。

**エリアマップ構成**：
```
町（防具屋・武器屋が機能／宿屋で全回復）
  │
砂漠（フィールド。暑さでスタミナが減る、といった環境ギミックを検討）
  │
オアシス（休憩ポイント。回復・小さな会話イベント）
  │
ピラミッド（ミニダンジョン。宝箱に防具が入っていることがある）
  │
オークキング（ボス）
```

**ワールドマップとの接続**：1章の`FIELD_LANDMARKS`にある`desert-town-entrance`
（`chapter1Data.ts`、砂丘に囲まれた城アイコン）が、1章ボス撃破後に本物の入り口になる。

### 第3章：魔法の森（テーマ：魔法を使えるようになる）

**新要素**：魔法システム本格導入。「ほのお」「みず」「かぜ」の言霊をタイピングで
唱えると、それぞれ火球・水流・風刃として発動する（試練の塔の魔法発動システム
＝`battle/page.tsx`の`MagicType`を、ストーリーモードの「唱える」演出に寄せて拡張する
イメージ）。MP管理もここから始まる。

**物語の流れ**：
- 森の入り口で、妖精たちが「言霊は唱え方ひとつで魔法にもなる」ことを教えてくれる。
- 妖精の里で、実際に3属性の魔法を1つずつ練習で覚える（チュートリアル的な流れは
  1章のコトの誘導パターンを踏襲するとよい）。
- 「迷いの森」は文字通り道に迷いやすいエリアにし、1章で覚えた「言葉を集める」
  基本と魔法を組み合わせて進む。
- 古代神殿の番人が、力試しとして戦いを挑んでくる。

**エリアマップ構成**：
```
森入口
  │
妖精の里（魔法の使い方を教わる。MPの概念もここで説明）
  │
迷いの森（フィールド。分岐が多く、魔法や覚えた言葉を頼りに進む）
  │
古代神殿（ミニダンジョン。番人との力試し）
  │
フェンリル（ボス）
```

**ワールドマップとの接続**：1章の`magic-forest-view`（森テクスチャに囲まれた目印）が
2章ボス撃破後に本物の入り口になる。

### 第4章：謎の洞窟（テーマ：魔法によるマップ攻略）

**新要素**：魔法を「戦闘」だけでなく「マップの謎解き」に使う。特定の障害物は
対応する魔法を唱えないと先に進めない（炎でツタを燃やす／風で風車を回す／水で
溶岩を冷やす、の3パターンを基本形にする）。プレイヤーは初めて「考えて進む」
ことを求められる。

**物語の流れ**：
- 謎の青年が「この先に古い遺跡がある」とヒントをくれて、洞窟探索のきっかけになる。
- 分岐ごとに異なる魔法ギミックを配置し、3属性の魔法をまんべんなく使わせる。
- 古代遺跡で、失われた言葉にまつわる壁画・記録を見つける（後の章の伏線置き場に
  使える）。

**エリアマップ構成**：
```
入口
  │
分岐（炎/風/水、いずれかの魔法で道を切り替えるギミック部屋が複数）
  │
地下湖
  │
古代遺跡
  │
ドラゴン（ボス）
```

**ワールドマップとの接続**：3章ボス撃破後、ワールドマップに新しい目印
（`ancient-cave-entrance`のようなID）を追加する。位置は魔法の森からさらに奥
（例：フィールドの北側を拡張）に置く想定。

### 第5章：荒れ果てた町（テーマ：謎の青年の正体判明、回復魔法）

**新要素**：回復魔法、宿屋の強化（毒・呪い等の状態異常も治せるようになる）、
状態異常回復の概念。

**物語の流れ**（既存の伏線回収）：
- `stranger`（謎の青年）は1章から「夢に出てくる誰か」と同一人物という伏線が
  張ってある（`DREAM_STRANGER_PORTRAIT`）。ここで正体を明かす。
- 主人公「あなた昔から知ってるみたい。」／青年「そうかもね。」という会話の後、
  コトの記憶も戻り始める（コトも過去にこの町と関わりがあった、という示唆にする）。
- 崩壊した町の「英雄の間」で、かつてこの町を守った英雄（の記憶・亡霊）と出会い、
  回復魔法を授かる。ボスというより「対話で決着する」演出向き。

**エリアマップ構成**：
```
崩壊した町（宿屋が強化され、状態異常も治せるようになる）
  │
時計塔
  │
地下施設
  │
英雄の間
  │
英雄（対話中心の決着。回復魔法を授かる）
```

**ワールドマップとの接続**：4章ボス撃破後に新しい目印を追加。

### 第6章：毒の沼地（テーマ：毒によるマップ上の攻撃効果）

**新要素**：毒が戦闘内の状態異常だけでなく、**マップを歩いているだけでダメージを
受ける地形ハザード**として機能する（沼の特定マスに乗るとHPが減っていく、など）。
5章で覚えた回復魔法・状態異常回復が実戦で活きる。

**物語の流れ**：
- 沼の入り口で、毒の沼地がかつての戦場・墓地であったことが示唆される
  （魔王との戦いの歴史に関わる伏線）。
- 道中で毒を解除する魔法を新たに覚える。

**エリアマップ構成**：
```
沼入口
  │
毒沼（地形ハザード：特定マスで継続ダメージ、回復魔法で対処）
  │
廃神殿
  │
地下沼
  │
クラーケン（ボス）
```

**ワールドマップとの接続**：5章ボス撃破後に新しい目印を追加。

### 第7章：なぞをとけ（沈黙の塔。テーマ：情報収集＋総合ダンジョン）

**新要素**：新しい魔法やシステムの追加は無し。**今まで覚えた全ての魔法・
ギミック・タイピング力を総動員する「卒業試験」**の章。魔王大陸へ渡るための
情報（渡り方・弱点等）を集める、謎解き寄りの章にする。

**物語の流れ**：
- 塔の最上階には、最後まで生き残った英雄（または英雄の記録）がいて、
  魔王城への道を教えてくれる。
- 世界の言葉がなぜ封印されたのか、終盤に向けての核心的な伏線をここで
  少しずつ明かす。

**エリアマップ構成**（フロア制のダンジョン。例示された順序をそのまま採用）：
```
1F：炎で氷を溶かす
2F：風で橋を動かす
3F：回復魔法を使いながら進む（継続ダメージ床など）
4F：毒を解除して進む
5F：光を反射させて進む（8章の光魔法習得前でも解ける謎にするか、
    「まだ解けない扉」として残し8章クリア後に戻って回収できる隠し要素にしてもよい）
最上階：英雄との対話。ボス戦は無し
```

**ワールドマップとの接続**：6章ボス撃破後に新しい目印を追加。

### 第8章：魔王討伐（魔王大陸。テーマ：集大成、光魔法）

**新要素**：光魔法（最後の魔法）。今までの全システム（装備・魔法・ギミック・
長い単語のタイピング）を使い切る集大成の章。

**物語の流れ**：
- 荒野を抜けたところにある「魔族の町」は、単純な敵の町ではなく一癖ある
  ポジションにしてもよい（魔族にも事情がある、という深みを出せる）。
- 闇の神殿で光魔法を習得。魔王城の闇属性ギミックに対抗する手段になる。
- ラスボス戦では、今まで覚えた中で一番長い言霊を入力して決着をつける
  （`getWordPower`が文字数に比例するバランス設計と相性が良い）。
- クリア後、世界に言葉の力が戻る。エンディングで「はじまりの村」に戻ると、
  旅立ちの頃より人々が笑い、会話し、子どもたちが言葉を学んでいる景色に
  変わっている（`frontend/DESIGN.md` 7.1節の「言葉で世界が変わる」演出）。

**エリアマップ構成**：
```
荒野
  │
魔族の町（ひと癖ある関わり方にできると深みが出る）
  │
闇の神殿（光魔法を習得）
  │
魔王城
  │
ラスボス（一番長い言霊で決着）
```

**ワールドマップとの接続**：7章クリア後に最後の目印が本物の入り口になる。

## 3. 実装の進め方（提案）

1. まず2章「砂漠の町」から着手する（お金・ショップという、他の章にも波及する
   基盤システムを最初に固められるため）。
2. ショップUIの具体的な設計（何が買えるか、価格、防御力への反映式など）を
   このファイルに追記してから実装に入る。
3. 魔法の「唱える」演出（3章）は、既存の`battle/`の魔法発動システムを土台に、
   ストーリーモードのフィールド上でも発動できるように拡張する設計を別途詰める。
4. 各章とも、まずはワールドマップ側の目印追加（`FIELD_LANDMARKS`の差し替え）から
   着手すると、既存パターンの流用で早く形になる。
